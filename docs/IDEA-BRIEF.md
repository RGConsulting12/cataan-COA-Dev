# Idea brief: Catan COA Advisor

**Slug:** `cataan-coa-dev`  
**Date:** 2026-06-26  
**Author:** COA Dev team  
**Repository:** https://github.com/RGConsulting12/cataan-COA-Dev

## Elevator pitch

A local API that accepts a snapshot of an in-progress **Catan** game and returns **three ranked courses of action** (build, trade, play card, move robber, etc.) with short rationales. Recommendations are grounded in `docs/CATAN-OFFICIAL-RULES.md` and generated by **Ollama** (`qwen2.5`).

## Problem

Catan has many legal moves each turn; newer players and even experienced players miss high-value trades, city timing, robber placement, and road races. A rules-aware advisor helps without replacing human negotiation.

## Proposed solution

- **FastAPI** service with `POST /recommend`
- Structured **game state JSON** in, **3 COAs** out (rank, action type, details, rationale, rules citations)
- **Ollama** (`qwen2.5`) with rules doc in system context
- Optional simple UI later (out of MVP scope)

## Users

- **Primary:** Player at the table entering board state manually or via a future client
- **Secondary:** Developers testing COA quality against sample positions

## Must-haves (MVP)

1. Official rules reference markdown committed and loaded into LLM context
2. `GET /health` — API + Ollama reachability
3. `POST /recommend` — returns exactly 3 ranked COAs for valid game state
4. Pydantic schemas for request/response; pytest coverage
5. `.env` configuration for `OLLAMA_BASE_URL`, `OLLAMA_MODEL=qwen2.5`

## Non-goals

- Full digital Catan board / multiplayer game engine
- Computer vision of physical board
- Expansions (Seafarers, C&K) in v1
- Cloud deployment or auth in v1

## Stack preference

- [x] **FastAPI** (HTTP API)
- [ ] Streamlit (optional dashboard in a later slice)

## Data and sensitivity

- No PII; game state is local tabletop data only
- Data stays on machine (Ollama local inference)

## Integrations

- Ollama (`http://127.0.0.1:11434`)
- GitHub via Ensamble for delivery

## Success metrics

- Valid JSON response with 3 distinct, rules-legal COAs on sample game states
- Response under ~30s on local hardware with qwen2.5
